## 🚀 Как использовать

### Запуск приложения
```bash
python run.py
```

### Запуск тестов
```bash
# Все тесты
pytest

# С подробным выводом
pytest -v

# Только unit тесты
pytest tests/test_unit_auth.py tests/test_unit_crud.py -v

# Только integration тесты
pytest tests/test_integration.py -v
```

### Примеры API запросов

#### 1. Регистрация
```bash
curl -X POST "http://localhost:8000/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'
```

#### 2. Логин
```bash
curl -X POST "http://localhost:8000/api/auth/login" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=user@example.com&password=password123"
```

#### 3. Создание ссылки (с аутентификацией)
```bash
curl -X POST "http://localhost:8000/api/links" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"original_url":"https://github.com"}'
```

#### 4. Получение списка своих ссылок
```bash
curl -X GET "http://localhost:8000/api/users/me/links" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## ✨ Ключевые особенности тестов

1. **Unit-тесты изолированные** - используют фикстуры для создания временных объектов
2. **Integration-тесты полные** - тестируют цепочку операций (регистрация → логин → создание → редирект)
3. **SQLite для тестов** - используется в памяти, поэтому тесты быстрые
4. **Фикстуры Pytest** - все конфигурируется в `conftest.py`
5. **Проверка безопасности** - тесты проверяют изоляцию данных между пользователями
